Use Case — TMDL Critical Rules

Triple-slash = Description

/// immediately before a declaration sets the Description property. A blank line between /// and the declaration breaks the model. Use // for regular comments.

Tabs, Not Spaces

Indentation is semantic. Tab depth = nesting level. DAX expression bodies sit at depth 3 — one deeper than measure properties. Spaces will break the TMDL parser.

Name Quoting

Quote names with spaces, dots, or special characters: 'Total Revenue', '1) Period'. Never quote simple names: Product, _Measures.

lineageTag

Every object needs a unique lineageTag. Never invent one manually — generate as UUID. Never change existing lineageTags; Power BI uses them for identity tracking.

PBI_FormatHint

Don't remove PBI_FormatHint annotations — Power BI Desktop re-adds them automatically. Leave them alongside your formatString.

Last Resort Principle

Prefer Tabular Editor CLI, Power BI MCP server, or connect-pbid over direct TMDL editing. Direct editing has no DAX validation or referential integrity checks.
